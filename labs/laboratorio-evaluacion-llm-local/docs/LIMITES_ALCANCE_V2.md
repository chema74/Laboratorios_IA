# ️ LÍMITES Y ALCANCE V2 - LABORATORIO EVALUACIÓN LLM LOCAL

## Declaración de alcance técnico

---

## 1. Objetivo

Este documento define los límites de la V2 (Version 2 - Versión 2) del laboratorio de evaluación local de LLM (Large Language Model - Gran Modelo de Lenguaje).

Su función es evitar exageraciones, separar laboratorio de producto y mantener una narrativa técnica honesta.

---

## 2. Qué sí demuestra

El laboratorio demuestra:

- Una base local de evaluación de LLM.
- Organización de criterios de revisión.
- Scripts ejecutables localmente.
- Tests automatizados.
- Enfoque free-first y local-first.
- Ausencia de claves reales.
- Ausencia de APIs (Application Programming Interfaces - Interfaces de Programación de Aplicaciones) de pago obligatorias.
- Separación entre evaluación local y evaluación productiva.

---

## 3. Qué no demuestra

El laboratorio no demuestra todavía:

- Plataforma productiva de evaluación LLM.
- Benchmark masivo industrial.
- Evaluación con datasets reales sensibles.
- Observabilidad completa.
- Integración con trazas productivas.
- Evaluación humana formal a escala.
- Comparativa exhaustiva entre proveedores.
- Despliegue cloud.
- Producto SaaS (Software as a Service - Software como Servicio).

---

## 4. Riesgos propios de la evaluación LLM

La evaluación de LLM puede presentar riesgos como:

- Métricas demasiado superficiales.
- Confusión entre fluidez y calidad real.
- Sesgo en prompts o ejemplos.
- Falta de ground truth.
- Resultados no reproducibles.
- Evaluaciones difíciles de comparar.
- Exceso de confianza en puntuaciones automáticas.

La V2 reconoce estos riesgos y los limita a un laboratorio local controlado.
