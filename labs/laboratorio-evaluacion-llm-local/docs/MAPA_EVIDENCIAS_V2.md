#  MAPA DE EVIDENCIAS V2 - LABORATORIO EVALUACIÓN LLM LOCAL

## Evidencias revisables del laboratorio

---

## 1. Objetivo

Este documento define las evidencias que permiten revisar la V2 (Version 2 - Versión 2) del laboratorio de evaluación local de LLM (Large Language Model - Gran Modelo de Lenguaje).

La finalidad es que una empresa, consultor TIC (Information and Communication Technologies - Tecnologías de la Información y la Comunicación) o evaluador técnico pueda entender qué demuestra el repositorio y qué límites mantiene.

---

## 2. Evidencias existentes

El laboratorio cuenta con evidencias asociadas a:

- Documentación técnica.
- Decisiones de diseño.
- Scripts locales.
- Tests automatizados.
- Métricas o criterios de evaluación.
- Salidas reproducibles.
- Ejecución local sin servicios cloud obligatorios.

---

## 3. Evidencias V2 recomendadas

Las evidencias V2 refuerzan la lectura profesional mediante:

- Validación automatizada de estructura.
- Validación de documentos clave.
- Validación de scripts y tests.
- Informe ejecutivo local.
- Revisión de mojibake UTF-8.
- Guía de revisión ejecutiva.
- Registro de límites de alcance.

---

## 4. Evidencia técnica

La evidencia técnica debe apoyarse en:

- Scripts Python disponibles en `scripts/`.
- Tests automatizados disponibles en `tests/`.
- Documentación disponible en `docs/`.
- Salidas generadas localmente en `salidas/`.

Estas piezas deben mantenerse locales, reproducibles y sin dependencias externas obligatorias.

---

## 5. Evidencia profesional

El laboratorio debe poder presentarse como una demostración de:

- Evaluación controlada de respuestas de LLM.
- Comparación de criterios de calidad.
- Revisión trazable de resultados.
- Disciplina básica de pruebas.
- Separación entre experimentación local y evaluación productiva.

No debe presentarse como sistema definitivo de evaluación empresarial a gran escala.
