# ⚠️ LÍMITES Y ALCANCE V2 — LABORATORIO RAG CORPORATIVO LOCAL

## Declaración de alcance técnico

---

## 1. Objetivo

Este documento define los límites de la V2 (Version 2 – Versión 2) del laboratorio RAG (Retrieval-Augmented Generation – Generación Aumentada por Recuperación) corporativo local.

Su función es evitar exageraciones, separar laboratorio de producto y mantener una narrativa técnica honesta.

---

## 2. Qué sí demuestra

El laboratorio demuestra:

- Un flujo local de recuperación documental.
- Una base de trabajo para RAG corporativo.
- Organización de documentación técnica.
- Scripts ejecutables localmente.
- Tests automatizados.
- Enfoque free-first y local-first.
- Ausencia de claves reales.
- Ausencia de APIs (Application Programming Interfaces – Interfaces de Programación de Aplicaciones) de pago obligatorias.

---

## 3. Qué no demuestra

El laboratorio no demuestra todavía:

- Despliegue productivo.
- Escalado empresarial real.
- Integración con directorios corporativos.
- Control avanzado de permisos por usuario.
- Observabilidad completa.
- Seguridad productiva integral.
- Evaluación masiva con datasets reales.
- Integración con un LLM (Large Language Model – Gran Modelo de Lenguaje) cloud obligatorio.
- Producto SaaS (Software as a Service – Software como Servicio).

---

## 4. Riesgos técnicos propios de RAG

Un sistema RAG puede presentar riesgos como:

- Recuperación de contexto irrelevante.
- Fragmentación documental deficiente.
- Respuestas plausibles pero no suficientemente fundamentadas.
- Falta de control de permisos.
- Exposición accidental de información sensible.
- Evaluación insuficiente de calidad.
- Dependencia excesiva del modelo generativo.

La V2 debe reconocer estos riesgos y documentar qué parte está cubierta en local y qué parte quedaría para fases posteriores.

---

## 5. Alcance profesional

El laboratorio sirve como evidencia de portfolio para demostrar criterio técnico en:

- IA (Artificial Intelligence – Inteligencia Artificial) aplicada a documentación corporativa.
- Arquitecturas RAG.
- Automatización local.
- Validación de laboratorio.
- Documentación técnica.
- Separación entre prototipo, laboratorio y producto.
