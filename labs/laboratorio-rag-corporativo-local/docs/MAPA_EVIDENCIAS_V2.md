#  MAPA DE EVIDENCIAS V2 - LABORATORIO RAG CORPORATIVO LOCAL

## Evidencias revisables del laboratorio

---

## 1. Objetivo

Este documento define las evidencias que deben permitir revisar la V2 (Version 2 - Versión 2) del laboratorio RAG (Retrieval-Augmented Generation - Generación Aumentada por Recuperación) corporativo local.

La finalidad es que una empresa, consultor TIC (Information and Communication Technologies - Tecnologías de la Información y la Comunicación) o evaluador técnico pueda entender qué demuestra el repositorio y qué límites mantiene.

---

## 2. Evidencias existentes

El laboratorio ya cuenta con evidencias iniciales asociadas a:

- Arquitectura del flujo RAG.
- Decisiones técnicas.
- Guía de ejecución.
- Mapa de evidencias V1.
- Scripts locales.
- Tests automatizados.
- Datos sembrados o generados localmente.
- Demo ejecutable en entorno local.

---

## 3. Evidencias V2 recomendadas

Las evidencias V2 deben reforzar la lectura profesional del laboratorio mediante:

- Validación automatizada de estructura.
- Validación de documentos clave.
- Validación de scripts mínimos.
- Validación de tests.
- Informe ejecutivo local.
- Registro de ejecución.
- Cierre técnico de rama.
- Tag de cierre si procede.

---

## 4. Evidencia documental

La documentación debe permitir responder:

- Qué problema empresarial representa el laboratorio.
- Qué flujo técnico implementa.
- Qué scripts se pueden ejecutar.
- Qué salida se espera.
- Qué riesgos tiene un sistema RAG.
- Qué no está cubierto por esta versión.

---

## 5. Evidencia técnica

La evidencia técnica debe apoyarse en:

- `scripts/comprobar_salud.py`
- `scripts/sembrar_datos.py`
- `scripts/ejecutar_demo.py`
- Tests incluidos en `tests/`
- Validaciones futuras V2 si se incorporan.

Estas piezas deben mantenerse locales, reproducibles y sin dependencias externas obligatorias.

---

## 6. Evidencia empresarial

El laboratorio debe poder presentarse como una demostración de:

- Recuperación documental corporativa.
- Respuestas con apoyo en contexto.
- Trazabilidad de información.
- Reducción de dependencia de búsqueda manual.
- Base para sistemas internos de consulta documental.

No debe presentarse como producto final listo para producción.
