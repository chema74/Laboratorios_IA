# ️ LÍMITES Y ALCANCE V2 - LABORATORIO OBSERVABILIDAD Y COSTES IA

## Declaración de alcance técnico

---

## 1. Objetivo

Este documento define los límites de la V2 (Version 2 - Versión 2) del laboratorio de observabilidad y costes de IA (Artificial Intelligence - Inteligencia Artificial).

Su función es evitar exageraciones, separar laboratorio de producto y mantener una narrativa técnica honesta.

---

## 2. Qué sí demuestra

El laboratorio demuestra:

- Una base local de observabilidad aplicada a IA.
- Organización de señales operativas.
- Revisión de costes o uso.
- Scripts ejecutables localmente.
- Tests automatizados.
- Enfoque free-first y local-first.
- Ausencia de claves reales.
- Ausencia de APIs (Application Programming Interfaces - Interfaces de Programación de Aplicaciones) de pago obligatorias.

---

## 3. Qué no demuestra

El laboratorio no demuestra todavía:

- Plataforma productiva de observabilidad.
- Integración real con proveedores cloud.
- FinOps empresarial completo.
- Monitorización en tiempo real de producción.
- Alertado operativo avanzado.
- Control presupuestario real con facturación.
- Observabilidad distribuida completa.
- Despliegue cloud.
- Producto SaaS (Software as a Service - Software como Servicio).

---

## 4. Riesgos propios

La observabilidad y costes de IA pueden presentar riesgos como:

- Métricas incompletas.
- Costes estimados frente a costes reales.
- Señales operativas insuficientes.
- Falta de trazabilidad por usuario o caso de uso.
- Falta de alertas.
- Lecturas ejecutivas demasiado simplificadas.
- Confusión entre simulación local y operación real.

La V2 reconoce estos riesgos y los limita a un laboratorio local controlado.
