# Decisiones Técnicas V1

## 1. Datos sintéticos con PII ficticia

Se usan datos sintéticos para permitir demostración pública del flujo, preservar privacidad real y repetir pruebas de forma controlada.

## 2. Reglas heurísticas locales

La detección y validación se implementan con reglas heurísticas para mantener trazabilidad, coste cero operativo y comportamiento determinista en entorno de laboratorio.

## 3. Sin APIs externas

No se usan APIs externas para evitar exposición de datos fuera del entorno local y reducir dependencia de proveedores.

## 4. Valor de anonimizar y seudonimizar

La anonimización reduce identificación directa y la seudonimización conserva continuidad operativa sin exponer identidad original.

## 5. Valor de minimizar y validar salidas

La minimización limita datos compartidos y la validación de salida ayuda a detectar fugas residuales antes de reutilizar resultados.

## 6. Límites frente a auditoría real

Este laboratorio no sustituye una auditoría legal, organizativa o técnica real de privacidad, ni certifica cumplimiento RGPD.

## 7. Enfoque de portfolio técnico

La V1 prioriza simplicidad, reproducibilidad y evidencia documental para aprendizaje y demostración técnica.
