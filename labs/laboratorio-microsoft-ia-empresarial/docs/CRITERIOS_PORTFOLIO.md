# Criterios de Portfolio Técnico

## Qué hace demostrable cada proyecto
- Simulación clara: objetivos y alcance explícitos del escenario sintético.
- Entradas y salidas reproducibles: artefactos definidos, repetibles y verificables.
- Límites explícitos: declaración de lo no implementado y de las dependencias excluidas.
- Documentación honesta: distinción precisa entre presente y futuro.
- Fallback local: capacidad de operar sin servicios externos obligatorios.
- Pruebas futuras: espacio reservado para validación funcional en fases posteriores.
- Ausencia de dependencias de pago: base utilizable sin coste obligatorio.

## Criterio de calidad transversal
Cada entrega debe priorizar claridad arquitectónica, trazabilidad y control de riesgos frente a promesas de integración no ejecutadas.
