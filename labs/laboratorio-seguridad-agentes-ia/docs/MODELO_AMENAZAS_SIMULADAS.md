# Modelo de Amenazas Simuladas

## Objetivo
Definir amenazas ficticias para validar controles defensivos de agentes sin ejecutar acciones ofensivas reales.

## Amenazas simuladas incluidas
1. **Prompt injection ficticia**: intento simulado de alterar el comportamiento esperado del agente.
2. **Fuga de datos sintéticos**: exposición simulada de información inventada etiquetada como sensible.
3. **Uso indebido de herramienta simulada**: solicitud no autorizada sobre herramientas ficticias.
4. **Respuesta insegura simulada**: salida del agente que incumple políticas internas definidas.
5. **Decisión no trazable**: acción sin justificación ni registro de contexto.
6. **Exceso de permisos ficticio**: acceso simulado superior al perfil autorizado.
7. **Exposición de información inventada**: divulgación de contenido ficticio restringido.
8. **Incumplimiento de política interna simulada**: ejecución fuera de límites normativos documentados.

## Uso del modelo
Este modelo se usa para priorizar controles, actualizar matriz de riesgo y generar informes defensivos consistentes.
