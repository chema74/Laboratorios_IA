# Resumen Web Portfolio

## 1) Título corto para web
Laboratorio de Empresa Sintética para IA Empresarial

## 2) Subtítulo breve
Entorno local gratuito para generar datos, eventos, documentos, crisis y escenarios sintéticos con fines de validación técnica reproducible.

## 3) Descripción corta para tarjeta web
Repositorio técnico local que integra 10 módulos para construir una empresa sintética completa.  
Permite generar evidencias trazables sin datos reales ni dependencias de pago obligatorias.  
Incluye validación por tests y demo narrativa final ejecutable en local.  
Orientado a portfolio profesional, no a operación productiva.

## 4) Descripción larga para página de proyecto
Este proyecto resuelve la necesidad de demostrar arquitectura, integración modular y trazabilidad técnica en un contexto empresarial simulado, sin exponer datos sensibles ni depender de entornos productivos.  
Evita datos reales para mantener seguridad, reproducibilidad y facilidad de revisión técnica en entorno local.  
Demuestra capacidades de diseño de flujos sintéticos de extremo a extremo: generación de datos base, simulación operativa, documentación derivada, evaluación comparativa y consolidación narrativa final.  
Se organiza en 10 proyectos internos especializados que se conectan como laboratorio integrador V1.  
Genera evidencias en carpetas de `datos_ejemplo/`, junto con guiones y expedientes técnicos reutilizables para revisión profesional.  
Es útil para evaluar competencias de ingeniería aplicada: estructuración modular, consistencia documental, control de alcance, reproducibilidad y lectura crítica de resultados en condiciones no productivas.

## 5) Capacidades demostradas
- Generación de datos sintéticos.
- Simulación de eventos de negocio.
- Fábrica documental sintética.
- Escenarios de prueba para agentes.
- Simulación de crisis.
- Revisión humana simulada.
- Gemelo digital operativo ligero.
- Privacidad con datos sintéticos.
- Comparación agente/proceso.
- Demo narrativa final.

## 6) Estado técnico verificable
- 10 proyectos internos implementados.
- 70 tests pasando.
- Ejecución local reproducible.
- Demo narrativa final disponible.
- Tag previsto o creado: `v1-local-integradora`.

## 7) Comandos de validación
```powershell
python -m pytest .\proyectos -q
python .\proyectos\10-demo-narrativa-empresa-completa\ejecutar_demo.py --seed 42 --dias 7
```

## 8) Límites honestos
- No hay datos reales.
- No hay cliente real.
- No hay IA real ejecutándose.
- No hay API productiva.
- No hay dashboard productivo.
- No hay integración real con Google Workspace.
- No hay integración real con Microsoft 365.
- No hay auditoría legal real.
- No hay benchmark real.
- No hay sistema productivo.

## 10) Uso recomendado en portfolio
En entrevista o revisión técnica, presentar este proyecto como laboratorio local integrador para demostrar criterio de ingeniería y capacidad de orquestación documental/técnica sin sobredimensionar alcance.  
El enfoque recomendado es mostrar primero validación objetiva (tests y demo final), luego arquitectura por módulos y, por último, límites explícitos para diferenciar claramente una V1 técnica de un sistema productivo.
